Each pro on a job has a selectionStatus:

Pro found the job themselves
offer
Pro sent an offer/quote
proRejected
Pro declined the request

What shows when

ElementCondition
Status Always. Shows real backend status, never overridden.
Offers badge When there are opportunity pros whose offers haven't been accepted.
selectionStatus = "opportunity" or "offer" + proOpportunityOfferSentAt + !opportunityAcceptedAt
Leave Review When any pro has selectionStatus = "offer" and review = null
All three are independent. An expired job can show Offers + Leave Review simultaneously. Status never changes based on offers or reviews pending — it always reflects the backend value.

How a Job Becomes "Completed"

Only one path:

  • Customer taps Leave Review
  • Fills out the review form
  • Selects "Yes" for "Did this Pro complete the job?"
  • Submits → POST /reviews with proCompletedJob: true
  • Backend listener sets job.status = "completed"

If the customer selects "No", the job status stays unchanged.
